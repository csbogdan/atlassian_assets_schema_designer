Phase 7 • External import workflow

Design, validate, compare, and deploy Atlassian Assets import documents.

JSM Assets Schema Designer is a production-grade Next.js application for editing Atlassian external import schema-and-mapping JSON documents. It combines a visual schema workbench, mapping coverage tools, layered validation, semantic diff, staged deletions, project collaboration, and live integration tools for import sources.

Server-backed Projects, sharing, versions, baselines, environments, and changelog storage
Primary audience Teams maintaining JSM Assets schemas, import mappings, and deployment workflows at production scale.
Core document AssetsImportDocument with schema.objectSchema, optional statusSchema, and mapping.objectTypeMappings.
Design goal Keep business rules in pure domain code, not in React components, and make large schemas workable without raw JSON-only editing.
Product scope

What the application does

The product covers the full external import lifecycle: import or open a project, inspect schema and mappings, validate, edit, compare against saved or live state, and push changes back to Atlassian.

Projects
Disk-backed collaboration

Authenticated workspaces with versions, baselines, activity, sharing, export, staged deletions, and environments.

Schema
Tree, graph, refs

Object type hierarchy exploration, schema detail editing, bulk attribute operations, and reference visualization.

Validation
Structured diagnostics

Layered diagnostics with JSON pointers, worker-backed execution, rule toggles, deferred findings, and quick navigation.

Operations
Live integration

Import from JSM, push mapping, remote diff, schema export, delete tools, GUID replacement, environment push, and icon sync.

Key workflows

Typical usage path

These are the flows the application is optimized around.

Schema maintenance

Open a project, inspect object types in the Schema Explorer, stage risky deletions, update attributes, validate, save a version, then push to an import source.

Change review

Capture a baseline, make edits, compare against the baseline or remote live document, review semantic and impact findings, then export a report or changelog.
