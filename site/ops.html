Deployment

GitHub Pages pipeline

The docs site is a standalone static artifact published directly from the site/ folder. This keeps the publishing path independent from the Next.js runtime build.

Source site/

Contains HTML entry points, shared CSS, shared JS, screenshots, diagrams, and a .nojekyll marker.

Workflow .github/workflows/docs-pages.yml

Configures Pages, validates the docs entry point, uploads the static artifact, and deploys to the GitHub Pages environment.

Flow

Deployment sequence

flowchart LR
  A[Push to main or master] --> B[GitHub Actions workflow starts]
  B --> C[Checkout repository]
  C --> D[Configure Pages]
  D --> E[Validate site/index.html]
  E --> F[Upload Pages artifact from site/]
  F --> G[Deploy to github-pages environment]
  H[workflow_dispatch] --> B
Workflow detail

What gets triggered

  • Pushes to main or master when files in site/** or the workflow file change.
  • Manual runs via workflow_dispatch.
  • Pages deployment permissions: contents: read, pages: write, id-token: write.
  • Concurrency control prevents overlapping deployments.
Local review

Previewing docs locally

Use the package script added for docs preview:

npm run docs:serve

The script serves the static site from site/ so design and copy changes can be reviewed before push.

Repository setup

Required GitHub setting

Enable Pages from Actions

In the repository Pages settings, choose deployment from GitHub Actions. Without that, the workflow can complete but the site will not be published.
