Route surface

API routes

The application exposes project routes for persistence and collaboration, plus tool routes that proxy Atlassian operations or execute server-side utilities.

Project routes /api/projects and /api/projects/[id]
  • GET /api/projects lists projects visible to the current user.
  • POST /api/projects creates a new disk-backed project.
  • GET /api/projects/[id] loads a single project if readable.
  • PUT /api/projects/[id] saves the full project payload.
  • PATCH /api/projects/[id] updates status, global share, shareWith, or unshareWith.
  • DELETE /api/projects/[id] deletes the project for admins only.
Project helpers Export and changelog routes
  • GET /api/projects/[id]/export returns the effective document with staging applied.
  • GET /api/projects/[id]/changelog exposes project change history data.
  • POST /api/admin/claim-projects supports ownership administration for legacy projects.
  • POST /api/admin/share supports user-sharing related admin flows.
Tool routes

Operational endpoints

Route Purpose Notes
POST /api/tools/import-from-jsm Discover import source and fetch schema-and-mapping Accepts bearer auth and can return discovery-only data or the full document.
POST /api/tools/push-mapping PUT or PATCH a mapping document to Atlassian Supports async mode.
GET /api/tools/mapping-progress Poll async progress Uses resource IDs returned by async push.
GET /api/tools/config-status Read import source status Returns IDLE, RUNNING, DISABLED, or MISSING_MAPPING.
POST /api/tools/remote-diff Fetch live remote document for comparison Feeds the diff view's remote source.
POST /api/tools/push-to-env Push current schema using a named environment token Discovers the mapping URL first, then performs the write.
POST /api/tools/export-schema Export schema plus icon information from Atlassian Supports dry-run style workflows from the UI.
POST /api/tools/delete-object-types Delete live object types from a schema High-risk destructive route used by the tool panel.
POST /api/tools/sync-icons Copy icon assignments between schemas Supports dry-run and matching controls.
Auth Credential-based access

Credentials provider auth checks email and password, applies lockout policy, and records login outcomes in the database.

Permissions Effective permission evaluation

Route handlers compute read, write, and admin rights from owner ID, global flag, and share list before serving project data.
