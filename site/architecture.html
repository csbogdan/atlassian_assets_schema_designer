Core rules

Architecture

  • Business logic belongs in src/domain.
  • React components stay thin and mostly orchestrate view state.
  • The Zustand store is the application state hub and mutation gateway.
  • Validators and analyzers return structured Diagnostic[].
  • Heavy validation work runs off the main thread.

Pure business logic

Domain model and modules

The central type is AssetsImportDocument. It combines schema definitions and mapping definitions and is the single document shape used throughout parsing, validation, transformation, diffing, and export.

Application state

Store and state model

documentStore.ts centralizes current document state, raw JSON, diagnostics, revisioning, project metadata, environment definitions, staged deletions, undo/redo, and active view state.

  • Document state: parsed document, raw JSON, selected object type, selected mapping, focused JSON path.
  • Validation state: diagnostics, validation pending flag, rule config, deferred diagnostics.
  • Project state: current disk project ID, versions, baselines, activity, revision, sync timestamp, sharing-derived project lists.
  • Transformation actions: update document, apply safe autofix, stage/unstage/commit deletions, save version, load/save project.
Performance path

Validation worker

sequenceDiagram
  participant UI as UI edit
  participant Store as documentStore
  participant Hook as useValidationWorker
  participant Worker as validation.worker.ts
  participant Domain as validateDocument

  UI->>Store: update document
  Store->>Hook: revision changes
  Hook->>Hook: debounce
  Hook->>Worker: post effective document
  Worker->>Domain: run validators
  Domain-->>Worker: diagnostics
  Worker-->>Hook: latest result only
  Hook-->>Store: setDiagnosticsFromWorker

The effective document sent to the worker already accounts for staged deletions, which keeps validation aligned with what would actually be exported or pushed.

Persistence

Projects and persistence

  • Projects are stored as JSON files under .jsm-projects.
  • Stored payload includes document, versions, baselines, activity, validation config, project settings, staged deletions, sharing, and environments.
  • Export endpoints apply staging before returning the document.
  • Optimistic concurrency uses timestamps to detect out-of-session changes before overwriting.
Security model

Authentication and permissions

  • Auth uses NextAuth credentials provider with bcrypt password verification.
  • Successful and failed login attempts are recorded in the database.
  • Permissions are derived from owner ID, global flag, and explicit share list.
  • Owners have admin rights. Global visibility expands read access. Explicit sharing expands access based on current permission logic.
